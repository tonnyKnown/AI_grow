# 天气查询格式化技能

此技能提供完整的天气查询工作流程，并输出格式化结果。

## 工作流程步骤

1. **查询当前天气**（当用户询问当前天气时）
   - 使用工具: `weather_query`
   - 参数: `city` (城市名称，如 "Beijing", "Shanghai", "北京")

2. **查询天气预报**（当用户询问预报时）
   - 使用工具: `weather_forecast`
   - 参数:
      - `city`: 城市名称
      - `days`: 1-5 (预报天数)

3. **格式化响应**
   - 解析天气工具返回的 JSON 结果
   - 格式化为易读的天气报告，包含:
      - 城市名称和查询时间
      - 温度（当前或预报范围）
      - 天气状况（晴/多云/雨/雪等）
      - 湿度和风速
      - 相关建议

## 响应格式

```
┌─────────────────────────────────────┐
│        🌤️  天气查询报告            │
├─────────────────────────────────────┤
│  📍 城市: 北京                     │
│  ⏰ 时间: 2024-01-15 14:30         │
├─────────────────────────────────────┤
│  🌡️ 温度: 15°C                     │
│  ☁️ 天气: 晴朗                      │
│  💧 湿度: 45%                       │
│  🌬️ 风速: 12 km/h                  │
├─────────────────────────────────────┤
│  💡 建议: 适宜户外活动              │
└─────────────────────────────────────┘
```

## 使用示例

**用户**: "北京今天天气怎么样？"
**技能响应**: 使用 `weather_query`，参数 `city="北京"`，然后格式化结果

**用户**: "上海未来3天天气预报"
**技能响应**: 使用 `weather_forecast`，参数 `city="上海"`, `days=3`，然后格式化结果

## 注意事项

- 如果用户未指定城市，请先询问
- 如果 API 返回错误，提供友好的错误提示
- 根据天气情况给出建议（如"记得带伞"、"注意添衣"）
